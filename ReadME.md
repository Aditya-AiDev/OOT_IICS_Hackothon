README: OTT Movie Streaming ETL PipelineTeam: Data PiratesMembers:Ritika SrivastavaAyushi AgrawalArindam SharmaAditya VermaProject OverviewThis project establishes an automated ETL (Extract, Transform, Load) pipeline for an OTT Movie Streaming platform. The architecture follows a multi-layered Medallion approach to ensure data quality, moving from raw ingestion to actionable KPI outputs.Pipeline Stages1. Pre-Processing & Ingestion (Bronze Layer)The pipeline begins by truncating all staging tables to ensure a clean slate for every run. Data is ingested from CSV sources into the staging area (STG).Watch Events: Ingesting both clean and dirty activity logs.Master Data: Ingesting Movie and User master records.[Insert Screenshot: Bronze Layer Ingestion]2. Data Cleansing (Silver Layer)Dirty events are processed through 7 specific validation rules using Expression, Filter, and Router transformations.Valid Records: Directed to TGT_CLEAN.Dead Ends: Failed rules and duplicate events are routed to TGT_REJECTED and TGT_DUPLICATE respectively.[Insert Screenshot: Silver Layer Cleansing]3. Data Modeling (Gold Layer)Clean data is transformed into a Star Schema for optimized querying.Dimensions: DIM_MOVIE and DIM_USER.Fact Table: FACT_WATCH_HISTORY is created using Joiners and Lookups to calculate derived fields like completion percentage and engagement scores.[Insert Screenshot: Gold Layer Modeling]4. Analytics & KPI OutputsThe final stage generates business-critical insights:Category Metrics: Monthly popularity (sum of watch time) and trends (top 3).User Metrics: Engagement scores and behavioral patterns by category.Geographic Insights: City-wise viewership summaries.[Insert Screenshot: KPI Outputs/Dashboards]
